## What changed this session (Mobile Native + PWA Rebuild)

- **Instant overlays:** `More`, `Add to calendar`, booking sheet, and buy
  options are client state via a History-API `OverlayProvider` ‚Äî no more
  `router.replace(?more=1)` / `?addToCalendar=1` / `?buyMenu=1` RSC refetch.
  Back button closes the top sheet.
- **Instant tabs:** mobile tab links prefetch; a `useLinkStatus` dot shows
  pending; mobile `loading.tsx` now uses content skeletons (book/home tailored)
  instead of a bare spinner.
- **Glass restored on iPhone:** removed the blanket `@media (hover:none)`
  flatten; only accessibility fallbacks (`prefers-reduced-transparency`,
  `prefers-contrast`) flatten now. Directional pager slide re-enabled on touch.
- **One Sheet primitive:** `src/components/ui/sheet.tsx` (focus trap, scroll
  lock, Escape, back-dismiss, safe-area) backs all mobile overlays. Remaining
  Radix mobile dialogs (booking detail/cancel/walk-on/recurring, family,
  skip-session, cancel-membership, coach sub-request) migrated via a drop-in
  `sheet-dialog` API.
- **No double-fire:** removed `onPointerDown`+`onClick` pairs on tab bar,
  segmented control, roll call, booking sheet, add-to-calendar.
- **Installable PWA:** `app/manifest.ts`, icons in `public/icons`, Apple meta,
  `public/sw.js` (network-first navigations, offline fallback, no API/auth
  caching) + `ServiceWorkerRegister` (production only).

Report failures in chat ‚Äî we'll fix and re-run E2E for each.
